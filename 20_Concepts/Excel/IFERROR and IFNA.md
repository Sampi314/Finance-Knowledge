# IFERROR and IFNA

> **TL;DR.** IFERROR and IFNA are logical functions used to trap and handle errors generated by other formulas, returning a custom result instead of an error value.

## When you'd use this

You would use these functions when you anticipate a formula might return an error due to missing data or invalid calculations (like dividing by zero). They are commonly wrapped around lookup functions like VLOOKUP to handle "not found" scenarios gracefully without showing an error to the user. They make spreadsheets look cleaner and prevent errors from propagating into downstream calculations.

## The 30-second version

Both functions intercept formula errors and return an alternate value you specify (like a zero or a blank string `""`). IFERROR is a "blunt instrument" that catches any type of error (e.g., `#N/A`, `#VALUE!`, `#REF!`, `#DIV/0!`, `#NAME?`). IFNA is more specific and only traps the `#N/A` error (typically returned when a lookup fails). Because IFERROR can unintentionally hide serious errors like a misspelled function name (`#NAME?`), it is often better to use IFNA when you specifically just want to manage `#N/A` errors.

## The full explanation

The IFERROR function checks a value or formula and returns a standard result if no error is detected, or a custom result if an error is found. Before IFERROR was introduced, users had to combine nested IF statements with ISERROR. IFERROR checks for all errors: `#N/A`, `#VALUE!`, `#REF!`, `#DIV/0!`, `#NUM!`, `#NAME?`, or `#NULL!`.

The IFNA function, introduced in Excel 2013, is similar but is designed solely to handle `#N/A` errors. When a function returns an `#N/A`, it typically indicates that a value is not available or not found (common with functions like VLOOKUP, MATCH, and HLOOKUP). Unlike IFERROR, IFNA ignores other errors, meaning it will not accidentally mask a different, more serious problem.

## Formula

`=IFERROR(value, value_if_error)`
`=IFNA(value, value_if_na)`

## Worked example

To handle a division by zero error (`#DIV/0!`) that occurs when a quantity field is empty or zero, you can use:
`=IFERROR(C5/D5, 0)`

To handle a missing lookup value gracefully using `IFNA` instead of `IFERROR`:
`=IFNA(VLOOKUP(H5, $B$5:$C$16, 2, FALSE), "Not found")`

## Excel and modeling notes

When handling lookup errors, it is usually recommended to use IFNA instead of IFERROR. For example, if a function name is misspelled (e.g., `ZLOOKUP`), Excel returns the `#NAME?` error. IFERROR will catch this error and hide the problem, whereas IFNA will let the `#NAME?` error show, making debugging easier.
Also, if the custom result value is supplied as an empty string (`""`), the cell will appear blank when an error is detected.

## Pitfalls and gotchas

- IFERROR catches all errors, which can inadvertently hide syntax errors like `#NAME?` caused by mistyped formulas.
- If the value being checked is empty, it is evaluated as an empty string ("") and not an error.
