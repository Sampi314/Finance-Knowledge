Power BI Blog: More on the Decomposition Tree Visualisation
===========================================================

1 April 2021

_Welcome back to this week’s edition of the Power BI blog series. This week,_ _we look some more_ _at the Decomposition Tree visualisation._

Last week we created a simple decomposition tree that looked like this:

![](https://sumproduct.com/wp-content/uploads/2025/05/e774d10cbbb9450fc45efbe51abdf434-225.jpg)

This week, we are going to expand on it.

**Adding Dimensions**

We can add another dimension of analysis to this data, by adding a **Year** slicer to the report page.

![](https://sumproduct.com/wp-content/uploads/2025/05/f32e5a15e2cf9c3e4d2d058458ce054d-286.jpg)

As we toggle through the years, notice that the order of **Countries** displayed in that column may be reordered:

![](https://sumproduct.com/wp-content/uploads/2025/05/f1140ff857fc3b6f5f97a6a24f4a6fc7-229.jpg)

**Locking Fields**

If we publish the report as it is, users will be able to delete the field breakdown and select their own order of fields instead:

![](https://sumproduct.com/wp-content/uploads/2025/05/72aa864d2854c6fefb1083fba0ab5792-216.jpg)

Further, they will be able to select the order of fields to breakdown:

![](https://sumproduct.com/wp-content/uploads/2025/05/36776d1da4d05b45bb5a5d09375f407c-183.jpg)

As the publisher, we can lock these fields down if we want. This is done by first hovering on the field in the Report view, and then click on the ‘Lock level’ icon.

![](<Base64-Image-Removed>)

For example, if we lock the **Country** and **State / Province Name** field, the end user will be unable to delete these two fields in their analysis:

![](<Base64-Image-Removed>)

They may only remove the **City** field and change it to **Color:**

![](<Base64-Image-Removed>)

That’s it for this week! Highlighting two more things to consider when using the Decomposition Tree. Join us next week for more on Power BI.
