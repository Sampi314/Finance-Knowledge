_On the final Friday of each month, we’re going to set Excel for you to puzzle over so that you can get your “Excel fix”. Challenge your office colleagues to see who can solve the puzzle quickest. There’s no prizes at this stage, you’re playing for bragging rights only!_

**_Data Filter Not Working_**

You need to download the [attached Excel file](https://sumproduct.com/assets/blog-pictures/2017/final-friday/july-17/sp-example-of-autofilter-error.xlsm)
 for this month’s challenge. Imagine you have data as follows:

![](https://sumproduct.com/wp-content/uploads/2025/05/61ff2a6a41f66870eaf80952888c1ec0.jpg)

In this example, I have certain “amounts” for each item, and each item has been ‘auto-numbered’ using the formula

**\=SUBTOTAL(103,$G$13:$G13)**

which is a cool formula to use for autonumbering **visible** data. For example, if I highlight all of the data, go to the ‘Data’ tab on the Ribbon and click on ‘Filter’ in the ‘Sort & Filter’ grouping (**ALT + A + T**), I can select ‘USA’ in the ‘Country’ filter as follows:

![](https://sumproduct.com/wp-content/uploads/2025/05/21773b105ba5467d1144218b2dca6199.jpg)

and I will get

![](https://sumproduct.com/wp-content/uploads/2025/05/0eb873463c4c0bda59bcf0d3322cc747.jpg)

See how the auto-numbering accommodates the filtering? Anyway, it’s a nice, useful trick but it distracts from this month’s conundrum…

**_The Challenge_**

In the [attached Excel file](https://sumproduct.com/assets/blog-pictures/2017/final-friday/july-17/sp-example-of-autofilter-error.xlsm)
, I want you to select ‘Australia’ from the ‘Country’ field:

![](https://sumproduct.com/wp-content/uploads/2025/05/7e2d1894f213f6b017d351c07bfe3041.jpg)

Huh? How come ‘USA’ is now part of the filtered ‘Australia’ subset? I assure you I selected all of the data for filtering – so what went wrong and how can you correct it?

Sound easy? Have a go. We’ll publish the answer in Monday’s blog. Have a great weekend!
