Power Pivot Principles: Introducing Slicers
===========================================

24 July 2018

_Welcome back to our Power Pivot blog. Today, we consider slicers in PivotTables created using Power Pivot._

We have touched on slicers briefly in [the ALLSELECTED blog](https://www.sumproduct.com/blog/article/power-pivot-principles/ppp-allselected)
, slicers display buttons representing the values in a field that we can select to filter the PivotTable data. This field does not need to be in the PivotTable explicitly.

Below is an example of a slicer used:

![](https://sumproduct.com/wp-content/uploads/2025/05/5a4816ee173ed4769ab1e7eca735a6fe.jpg)

We can use multiple slicers on the same PivotTable. For instance, we can insert the **SalesTerritoryKey** slicer:

![](https://sumproduct.com/wp-content/uploads/2025/05/c011673a3ef913ef83759e20d044fbe8.jpg)

The PivotTable will update to display sales values from January to June, and from the Territory key of 6.

Of course, we can do all of this with ordinary PivotTables. The difference with Power Pivot is that we can insert slicers for fields contained in the different tables that are linked together in the data set.

Notice the ‘All’ tab on the ‘Insert Slicers’ dialog box?

![](https://sumproduct.com/wp-content/uploads/2025/05/5bfb76e1a853da099161ec4e94a64143.jpg)

Let’s try adding a slicer for the country:

![](https://sumproduct.com/wp-content/uploads/2025/05/684118aba46c22d951614ced895fba8a.jpg)

The PivotTable will update to the selections made on any of the slicers. We can add as many slicers as we want from any of the other tables in the data set, as long as those tables have proper relationships established! Isn’t that neat?

![](https://sumproduct.com/wp-content/uploads/2025/05/cbe4eaf1a89ffd5bdad1ba8e1668ca26.jpg)
