Power BI Blog: No More Q&A
==========================

_Welcome back to this week’s edition of the Power BI blog series.  This week, we review the forthcoming deprecation of Q&A, Microsoft’s legacy natural language tool for Power BI._

![](https://sumproduct.com/wp-content/uploads/2026/01/image-91.png)

Q&A, Microsoft’s legacy natural language tool, will be deprecated in December 2026, which is disappointing as we always find it better than its alternative (presently).  Moving forward, Microsoft advises you should use Power BI Copilot, which should provide a more advanced and integrated solution for querying data with natural language.

Microsoft has explained that the reason for the change is because Power BI Copilot now features an advanced natural language interface using generative AI, letting users ask questions, create visuals, and explore insights more accurately and more flexibly than with Q&A.

It is confusing having alternatives.  By consolidating our natural language capabilities into Copilot, the aim is to be able to:

*   reduce confusion between overlapping features
*   accelerate innovation and delivery of new and modern AI capabilities
*   provide a more consistent and intuitive experience across Power BI.

To support this transition, Microsoft states that they are reviewing the approach for Pro licenses.

The following will therefore be affected by the impending deprecation:

*   **Q&A experiences:** this is the feature that allows users to interact with their data via natural language.  This includes:
    
    *   **Q&A in reports**, which is available if the report author added a specific type of interactive visual called a Q&A visual or added a Q&A button
    
    *   **Q&A on a dashboard**, which appears as a question box or button on the dashboard.  Q&A is available at the top on dashboards in Power BI Service and at the bottom of the dashboard in Power BI mobile (Q&A virtual analyst in the mobile application)
    
    *   **Q&A in Power BI embedded analytics**, where developers may integrate Q&A into custom applications through APIs or embedded tokens  
          
        
*   **Q&A Setup:** this is a suite of tools aimed at helping report authors improve the Q&A experience.  This includes synonyms, linguistic relationships, row labels, hide from Q&A, teach Q&A, review questions and suggest questions.

The timeline is as follows:

*   **December 2025:** deprecation announcement (only slightly late then)
*   **December 2026:** Q&A experiences and Q&A Setup will be fully retired from Power BI Desktop and Service.  New creation of any Q&A experiences listed herein will no longer be permitted.  Existing Q&A visuals in reports, dashboards, mobile and embedded scenarios will no longer work or be available.

If you are using Q&A experiences already, you should consider transitioning to Copilot and Chat with your data experiences in Power BI.  Microsoft proposes the following alternatives:

*   Q&A in reports: use Copilot with Power BI reports
*   Q&A in Dashboards: use Copilot standalone experience
*   Q&A in Mobile: use Copilot in Power BI Mobile
*   Q&A Embedded: use Copilot for SaaS scenarios.  For PaaS, Copilot support should be arriving in the coming months.

If you are using Q&A Setup, you should also familiarise yourself with ‘Prep Data for AI’ and how it can replace Q&A Setup tooling.

To support this transition, Microsoft has admitted it is reviewing the approach for Pro licenses. Please stay tuned for updates in the coming months.
