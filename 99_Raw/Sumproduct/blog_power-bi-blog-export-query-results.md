Power BI Blog: Export Query Results
===================================

_Welcome back to this week’s edition of the Power BI blog series.  This week, we look at exporting query results._

![](<Base64-Image-Removed>)

‘Export Query Results’ in Power BI Desktop unlocks data portability and collaboration across Microsoft Fabric.  This feature empowers you to take transformed, cleaned data from Power Query and export it directly to trusted destinations, such as Dataflows Gen2, Lakehouses and other cloud stores, without relying upon complex workarounds or third-party tools.  This reduces the duplication of effort, accelerates data sharing and drives interoperability across Fabric workloads.

To enable this feature, select the Preview feature settings in Power BI Desktop.  Then, you may select ‘Export Query Results from the Power Query Ribbon,

![](<Base64-Image-Removed>)

select an online destination

![](<Base64-Image-Removed>)

and export your queries to this destination where you may choose to modify the Dataflow.

![](<Base64-Image-Removed>)

The result is improved data gravity in OneLake, streamlined refresh and monitoring capabilities, and a minimal-click experience that supports both new and existing destinations.  Ultimately, this feature should bridge the gap between desktop analytics and enterprise-scale data integration, enabling organisations to maximise the value of their data investments.
