Claude vs. Agent Mode: Battle of the AI Modelling Assistants
============================================================

**The Challenge**

Seconds away, round 1!! 

After the unprecedented feedback generated by our last article, we thought we should host a rematch for Claude for Excel as soon as possible.  Several of you asked how Claude and Copilot would interact in Excel.  Given Agent Mode is probably the better tool for comparison purposes (it generates formulae as does Claude, rather than provide results like Copilot), we decided to let them battle.

We gave Microsoft’s Excel Labs Agent Mode and the Claude Excel add-in the same task: build a clean, fully auditable Discounted Cash Flow (“DCF”) under strict standards.  Same prompt.  Same colour discipline.  Same rulebook, _i.e_. “no hardcode, no volatility, no circulars”.

Both got there — but by very different routes.

![](<Base64-Image-Removed>)

**_Agent Mode: The Deep Thinker_**

Agent Mode did not rush.  This might test the patience of those of you under immense time pressure.  Like a snail caught in rush hour traffic, it spent nearly four \[4\] minutes planning, outlining sheets, flagging risks, pre-building checks and several other aspects of the whole project plan routine.

![](<Base64-Image-Removed>)

The result was a very “by-the-book” model:

*   six \[6\] sheets, all cleanly separated
*   volume × price build for revenue (simple but tidy)
*   flat operating assumptions with a strict one formula per cell style
*   plenty of checks concerning Net Operating Profit After Tax (NOPAT), Free Cash Flow (FCF), Present Values, roll forward calculations, _etc._

You can download the Excel file that Agent Mode generated **[here](https://sumproduct.com/wp-content/uploads/2025/11/SP-Agent-Mode-Test.xlsx)
**.

![](<Base64-Image-Removed>)

It produced a reasonable model from an AI perspective, but there was one catch: you cannot interrupt the routine.  It thinks, then it acts.  If it picked the wrong approach, you would have to wait for it to finish before steering the task back on course.

**_Claude Add-in: The Collaborator_**

Claude works like a Senior Analyst who likes to chat whilst they build.  You see formulae appear.  You can jump in mid-flow.  If something looks off, you can correct the course direction immediately.

![](<Base64-Image-Removed>)

The model took a different route to the goal:

*   seven \[7\] sheets, including an executive summary
*   revenue starts at $100m with a declining growth path
*   margins improve over time (COGS easing down, SG&A tightening)
*   cross-sheet links used freely, with more consolidated formulae
*   pass / fail indicators were preferred for the basis of checks, rather than confirming sub-totals, _etc_.

You can download the Excel file that Claude Excel add-in generated **[here](https://sumproduct.com/wp-content/uploads/2025/11/SP-Claude-Test.xlsx)
**.

![](<Base64-Image-Removed>)

Claude came across as more user-friendly, visual and collaborative — but you should keep watch as it can go wrong.  Interestingly, if your credits run dry during the build phase, be prepared you might be waiting for something to happen needlessly, until you realise it has paused.

**_The Devil’s in the Details_**

Both tools produce institutional-grade formulae with direct references and transparent logic.

**Agent Mode approach:** This was a pure, one-step-per-line logic for easy auditing.  Each cell performs a single calculation and tells one story clearly.  It also handles external references more rigorously by creating a dedicated section for pulling numbers from other sheets, ensuring transparency and consistency.

![](<Base64-Image-Removed>)

**Claude Add-in approach:** Modellers might recognise their own habits here.  Formulae may be more efficient, combining steps where sensible and using extensive cross-sheet linking.  However, it often embeds external references directly within formulae (which can cause difficulty for comprehension) rather than maintaining a separate section, which can make auditing slightly less straightforward.

![](<Base64-Image-Removed>)

**_Quality Assurance_**

**Agent Mode:** it actively re-runs key calculations to confirm they reconcile correctly.  For example, it verifies relationships such as **NOPAT = EBIT × (1 – Tax)**, it checks free cash flow components, it reconciles present values and validates changes in working capital, _viz._

![](<Base64-Image-Removed>)

**Claude Add-in:** this add-in focuses on process verification through pass / fail indicators, testing calculation methods, valuation methodologies and the terminal value approach, for example.

![](<Base64-Image-Removed>)

**_When to Use Which?_**

**AGENT MODE**

You should use Agent Mode when you prefer hands-off execution and want to simply submit your request and walk away.  It is ideal if you need comprehensive upfront planning to ensure a well-structured model.  Choose this option when you require the cleanest structure and can allow three to four minutes for thoughtful output.  Agent Mode is also best suited for those who value systematic, self-correcting methodologies.

![](<Base64-Image-Removed>)

**CLAUDE FOR EXCEL**

Claude Add-in is the right choice when you want real-time collaboration and the ability to interact during the build process.  Select this tool when strict formatting compliance is essential and you require presentation-ready deliverables.  Claude Add-in is ideal if you appreciate the flexibility to redirect the build mid-process.

![](<Base64-Image-Removed>)

**_Word to the Wise_**

Both tools can produce basic DCF models saving some manual work.  The choice is not about which is “better”; it is about which fits your workflow and habits:

*   Agent Mode offers autonomous, self-correcting thoroughness
*   Claude Add-in offers collaborative agility and real-time insights.

However, whilst AI assistants are getting better, they are not yet ready to replace experienced modellers – and heavy reliance on them will require more checking, verification and auditing.

Just remember, no tool, however sophisticated, eliminates the need for understanding what you are building.  These assistants can construct formulae, but you still need to understand the transaction, the rationale, the financing and the alternatives.  They can build structure, but you still need to validate assumptions and the overall architecture.  They can save time, but they cannot replace judgment.
