Charts and Dashboards: Custom Axes – Part 3

Charts and Dashboards: Custom Axes – Part 3
===========================================

2 December 2022

_Welcome back to our Charts and Dashboards blog series._ _This week, we complete this method of customising our axes as we format the Error Bars to create customised gridlines._

Consider the following data:

![](https://sumproduct.com/wp-content/uploads/2025/05/85dcf885fccb30cae8ab850d78cd7f88.jpg)

These are **X** and **Y** values to be plotted on a Scatter chart. Sounds simple enough, right?

![](https://sumproduct.com/wp-content/uploads/2025/05/29c56efb227cde971fbafbc9c6dfff6d.jpg)

There we have it. But already we can see some issues. It’s hard to discern the exact value of each point as they sit between the gridlines. Let’s create custom axis to make these values easier to determine.

In [Part 1](https://sumproduct.com/blog/charts-and-dashboards-custom-axes-part-1/)
, we created two new data series, one to act as the **x**\-axis of our chart and one to act as the **y**\-axis.

![](https://sumproduct.com/wp-content/uploads/2025/05/aa626d51bae2c607039723130e72b3af.jpg)

We plotted them on our chart:

![](https://sumproduct.com/wp-content/uploads/2025/05/063329601ba44a34640cb1728eb2f307.jpg)

After changing the colour and adding and formatting the data labels, we were ready to customise the gridlines:

![](https://sumproduct.com/wp-content/uploads/2025/05/07810f583d7fee62234ab6fd6ecdac61.jpg)

[Last week](https://sumproduct.com/blog/charts-and-dashboards-custom-axes-part-2/)
, we removed the default gridlines and created ‘Error Bars’.

![](https://sumproduct.com/wp-content/uploads/2025/05/49260d2dd5dc9e621926f38740ab1906.jpg)

This added error bars in both the **x** and **y**\-directions:

![](https://sumproduct.com/wp-content/uploads/2025/05/003fd97fef6ac424e18c9814cae35869.jpg)

This week, we will format the ‘Error Bars’ to create our custom gridlines. For the **x**\-axis, we only want these to extend in the **y**\-direction. We can select the Error Bars along the **x**\-axis and delete them. We can then format the remaining Error Bars:

![](https://sumproduct.com/wp-content/uploads/2025/05/7870a3ff6397c69eac040e7b2df559f3.jpg)

We want these to extend only in a positive direction (Plus), with ‘No Cap’ to a ‘Fixed Value’ equal to the maximum value of our data. We can then repeat this process for the **y**\-axis:

![](https://sumproduct.com/wp-content/uploads/2025/05/0fa03b83b10b7afca996d998adf75803.jpg)

The graph is looking a little skewed within the Chart Area as the default axis Minimum and Maximum are still set to Auto, we can manually set the Minimum and Maximum of both axes to zero \[0\] and 40 respectively:

![](<Base64-Image-Removed>)

We have achieved our goal: we have custom axes and gridlines to make the data easier to interpret:

![](<Base64-Image-Removed>)

That’s it for this week. Come back next week for more Charts and Dashboards tips.
