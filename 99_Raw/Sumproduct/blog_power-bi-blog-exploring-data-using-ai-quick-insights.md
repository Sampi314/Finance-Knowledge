Power BI Blog: Exploring Data using AI – Quick Insights
=======================================================

_Welcome back to this week’s edition of the Power BI blog series.  This week, we explore Quick Insights in Power BI._

We live in an era where Artificial Intelligence (AI) is increasingly being used by businesses to turn their data into information that can support business decisions.  This week, we will explore a feature called ‘Quick Insights’.  This tool is designed for business users with minimal **DAX** knowledge.  Read on to learn how to leverage this AI-powered tool to efficiently explore and understand your data.

**_Technical Requirements_**

To replicate the examples that follow, your computer needs to meet these technical requirements:

*   a work or school account with access to [www.powerbi.com](http://www.powerbi.com/)
    
*   Power BI Desktop
*   Power BI Premium, F-SKU, Fabric Trial or a Premium Per User licence.

You can download the Power BI Desktop practice file along with its source data from [here](https://sumproduct.com/wp-content/uploads/2026/03/PBI-Blog-Quick-Insights-Resources.zip)
.

The ‘Quick Insights’ tool is an AI-powered Power BI feature that generates insights from a semantic model with very little user effort.  The ‘Quick Insights’ tool is available in Power BI Service.  Behind the scenes, the ‘Quick Insights’ tool uses machine learning algorithms to identify patterns, trends and other useful insights in the semantic model.  The prerequisites for using ‘Quick Insights’ are as follows:

*   the user must have at least a Contributor role in the workspace and Build permission on the semantic model
*   the semantic model must be set to Import mode for storage.  Models using DirectQuery mode are not compatible with the ‘Quick Insights’ feature.

Note also that PBI reports using Row Level Security (RLS) are not supported for ‘Quick Insights’ – even in Import mode.

Let’s go through the following steps to generate insights using the ‘Quick Insights’ feature:

First, sign in to Power BI Service, choose the appropriate workspace and locate the semantic model you wish to analyse
