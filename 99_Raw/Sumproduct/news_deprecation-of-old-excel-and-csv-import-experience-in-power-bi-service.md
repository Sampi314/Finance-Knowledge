Deprecation of Old Excel and CSV Import Experience in Power BI Service
======================================================================

Excel and CSV files provide some of the most common data sources for Power BI semantic models and reports.  They are used frequently to create reports in Power BI Service from the ‘Create’ page or in Power BI Desktop.

If you have used what Microsoft refers to as “the old experience” to import an Excel or CSV file from the ‘Create’ page in the Service, please take note of the following.  However, we emphasise that there is no impact for users who created semantic models in Power BI Desktop.

To notice which ones are affected, semantic models created by the old import files experience cannot be edited in the browser or downloaded and will have no options to schedule a refresh.  If your semantic model can be edited in the browser, downloaded or has a scheduled refresh option, don’t worry, you’re already using the new Excel and CSV connectors, and there will be no impact on your semantic model and reports.

Reports created using the old import Excel or CSV experience will stop refreshing from their source file as of **Friday 31 July 2026**.  There will be no workaround or fix to keep them updated.  These reports need to be recreated to continue updating. 

These reports can still be viewed and edited after this date, but the data will no longer synchronise with the source Excel file.

To prevent you inadvertently creating such circumstances, Microsoft is removing the old Excel and CSV import experience in Power BI Service, which is currently accessible through the ‘Create’ page, by Sunday 31 May 2026 _(why not straight away?)_.  The existing options for Excel and CSV, as well as ‘Get data’, will continue to be available.  You should start using the other options for Excel and CSV files.

![](https://sumproduct.com/wp-content/uploads/2026/02/image-159.png)

_The legacy / old Excel and CSV import option is outlined in red at the bottom center of the Power BI Service Create page._

In summary, the key dates to remember are as follows:

*   **Sunday 31 May 2026:** you will no longer be able to create new semantic models using the legacy import experience
*   **Friday 31 July 2026:** existing semantic models created with the legacy experience will stop refreshing.  Reports based upon these models will still open, but their data will no longer update.  To keep your reports current, recreate them using a supported import method
*   **Monday 31 August 2026:** existing semantic models created with the legacy experience will no longer be available for querying.  Reports bound to these semantic models will no longer open and will display an error message instead.
