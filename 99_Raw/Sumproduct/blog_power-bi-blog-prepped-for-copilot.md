Power BI Blog: Prepped for Copilot
==================================

_Welcome back to this week’s edition of the Power BI blog series._  _This week, we inform you that ‘Prepped for AI’ has had a name change._

![](<Base64-Image-Removed>)

In May 2025, Microsoft launched the capability to mark a model as ‘prepped for AI’, which we have detailed [previously](https://sumproduct.com/blog/power-bi-blog-copilot-search-improvements-in-power-bi/)
.  The name of this setting has now been updated to ‘approved for Copilot’.  I feel so much better now.

Enabling this setting in a semantic model applies three \[3\] main functionalities:

1.  The standalone Copilot agent doesn’t apply friction treatment to answers from that model
2.  Authors can turn on the administrator setting to only show approved items in the standalone Copilot experience
3.  Models marked as ‘approved for AI’ are boosted in the Copilot search experience.

After you prepare your semantic model for AI and it’s ready for use, you can mark your model ‘Approved for Copilot’ by following these steps:

*   Navigate to Power BI Service and find your semantic model
*   Select the Settings icon in the upper left corner
*   Expand the ‘Approved Copilot’ section.
*   Select the ‘Approved Copilot’ box, and then select the ‘Apply’ button.
