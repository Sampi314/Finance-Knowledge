This course focuses on using AI as a force‑multiplier across the project finance modelling workflow. It emphasises AI‑supported processes – documentation, review, assumption extraction, scenario commentary, and credit analysis – while maintaining PF standards for auditability and transparency.

### Learning outcomes for AI in Project Finance Modelling:

*   Explain where AI adds real value in PF modelling vs where it’s unsafe/unreliable
*   Apply a simple data-governance “traffic light” approach to prompts + documents
*   Use AI tools to document models, break down formulas, and support model review
*   Extract assumptions from “data room” style documents into a structured input sheet
*   Create first-draft credit/IC narratives from model outputs (with controls)
*   Leave with a prompt library + workflow playbook designed for PF teams

### Who is this for?

Project finance modellers, credit analysts, lenders/advisers, model reviewers, infra/energy finance teams.

Course Agenda
-------------

**Foundations of safe AI use in project finance:**

*   Governance frameworks, confidentiality controls, data classification.
*   Redaction and paraphrasing techniques.
*   Safe prompt design.

**Model documentation and understanding:**

*   Generating model descriptions from sheet lists.
*   Converting formulas into plain‑English.
*   Producing reviewer‑friendly documentation.

**Review & quality assurance:**

*   AI‑generated checklists.
*   Identifying logical inconsistencies and likely failure points.
*   Scenario‑based red‑flag detection.

**Assumption extraction:**

*   Turning transaction documents into structured input tables.
*   Source referencing and confidence grading.
*   Distinguishing extracted vs. model‑ready assumptions.

**Scenario commentary & IC materials:**

*   Executive summaries from model outputs.
*   Risk/mitigant tables.
*   Covenant headroom commentary.

**Conceptual prototype design:**

*   AI‑generated deal structures for conceptual PF arrangements.
*   Understanding AI limits: circularity, tax, accounting, cash sweeps

### Explicit Exclusions

*   No coding (Python, VBA, automation).
*   No quantitative AI modelling.
*   No market forecasting or price analytics.
