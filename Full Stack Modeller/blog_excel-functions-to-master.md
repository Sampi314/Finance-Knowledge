The Excel Functions every modeller needs to master
==================================================

Published [by Myles Arnott](https://www.fullstackmodeller.com/blog/author/myles-arnott)

Oct 3, 2023 9:58:25 AM

Discover the key Excel functions for spreadsheet modellers

Microsoft has been busy releasing lots of new and powerful functionality into Excel recently. The rate of change is pretty mind blowing. It's very easy to be distracted and overwhelmed by all this new shiny stuff.

For most modellers however it is much more important to focus on your core skills and ensure that you master them.

In this series of articles I am going to focus on four key areas that all modellers need to master before they move onto Excel's more advanced functionality:

*   The Excel functions that every modeller needs to master
*   Using Excel Tables to transform the way you model
*   What's so great about Pivot Tables anyway?
*   Why you should be modelling in line with a Best Practice Standard

Let's make a start with the key functions:

Building reports  

-------------------

A core part of any modeller's role is turning data into insightful information. Invariably this means creating charts and tables from a dataset. The go-to function here is [SUMIFS](https://www.fullstackmodeller.com/blog/how-to-use-the-sumifs-function)
.

The [SUMIFS](https://www.fullstackmodeller.com/blog/how-to-use-the-sumifs-function)
 function returns the total value for rows in a dataset that meet the specified criteria.

![ezgif.com-gif-maker](https://www.fullstackmodeller.com/hs-fs/hubfs/ezgif.com-gif-maker.gif?width=2220&name=ezgif.com-gif-maker.gif)

  

Mapping data  

---------------

In modelling we regularly need to add more information to our main dataset by looking up information from a reference (or mapping / dimension table). Here we turn to the [XLOOKUP](https://www.fullstackmodeller.com/blog/how-to-use-the-xlookup-function)
 function which "maps" data from one table into another based on a unique reference.

[XLOOKUP](https://www.fullstackmodeller.com/blog/how-to-use-the-xlookup-function)
 is a much improved version of existing lookup methods such as VLOOKUP, HLOOKUP and INDEX MATCH.

![ezgif.com-gif-maker-xlookup](https://www.fullstackmodeller.com/hs-fs/hubfs/ezgif.com-gif-maker-xlookup.gif?width=2220&name=ezgif.com-gif-maker-xlookup.gif)

Working with dates  

---------------------

As modellers we often find ourselves needing to transform dates within our Excel models. A common requirement is to turn a date into the last day of the month so that we can use that date for reporting. The [EOMONTH](https://www.fullstackmodeller.com/blog/how-to-use-the-eomonth-function)
 function does exactly that:

Working with Text
-----------------

Transforming text is another key skill for modellers. We regularly need to tidy text, extract elements of text or combine text together.

The key functions to learn to manipulate text are [TEXTSPLIT](https://www.fullstackmodeller.com/blog/how-to-use-the-textsplit-function)
 and [TEXTJOIN:](https://www.fullstackmodeller.com/blog/how-to-use-the-textjoin-function)

![TEXTSPLIT Rows](https://www.fullstackmodeller.com/hs-fs/hubfs/TEXTSPLIT%20Rows.gif?width=2544&height=912&name=TEXTSPLIT%20Rows.gif)

Note that TEXTSPLIT is a dynamic array function as it "spills" the results over multiple cells rather than just the one cell.

![TEXTJOIN](https://www.fullstackmodeller.com/hs-fs/hubfs/TEXTJOIN.gif?width=2004&height=912&name=TEXTJOIN.gif)

IF Statements
-------------

Last, and certainly not least is the humble, yet powerful, [IF](https://www.fullstackmodeller.com/blog/how-to-use-the-if-function)
 function.

The IF function returns a specified value based on the result of a logical (True/False) test.

For example if we would like to check if the value in A1 is the equal to 1 and if its is equal to 1 then we would like to return "Yes", if it isn't then we would like to return "No". The formula would be:  

#### \=IF(A1 = 1, "Yes", "No")

![if-function-image](https://www.fullstackmodeller.com/hs-fs/hubfs/if-function-image.png?width=978&height=801&name=if-function-image.png)
