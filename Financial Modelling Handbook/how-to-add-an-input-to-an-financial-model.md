# Core modelling skill 1: How to add an input

**Source:** https://www.financialmodellinghandbook.org/how-to-add-an-input-to-an-financial-model

There are two options for adding a new input.

1.  Create the input on the input sheet, then link to it.
2.  Create the input on the calculation sheet, then relocate it later.

I prefer the latter as it allows me to change the input and generally mess around with it without paging back and forward to the input sheet. We will not leave our model with the inputs scattered throughout the calculation sheets. They will all be on the input sheet by the end of the process.

Having inputs on the calculation sheets, even temporarily during the model build, causes some modellers to lose their minds. If this is you, I get it. I'm not judging you. It seems at first to go against all basic modelling principles. By all means, use method one.

I'm going to demo method 1. If you want to use method 2, you'd simply use the "Create a link" mechanic.

Step 1: Write the row label.
----------------------------

I always start with this step. That way, if I get interrupted or distracted and don't complete the process, I know what I was doing. I avoid the risk of stray unlabelled numbers in the model.

![](https://www.financialmodellinghandbook.org/content/images/public/images/ab207b0e-14b5-4431-b142-acb9c2e389cb_2190x730.jpeg)

It's worth spending some time thinking about the labels you use in their model. See "[What makes a poor row label](https://www.financialmodellinghandbook.org/what-makes-a-poor-row-label/)
".

Step 2: Mark the input
----------------------

Ctrl+Shift+i adds standard input formatting.

![](https://www.financialmodellinghandbook.org/content/images/public/images/5af27d0f-3af8-4f30-baca-38f70fe7e9ae_2228x702.jpeg)

Step 3: Enter the input.
------------------------

In this case, I'm entering 0.065 since my units will be USD.

Initially, this rounds to zero.

![](https://www.financialmodellinghandbook.org/content/images/public/images/3d2f9b7c-a68d-4a91-8da9-7352b05041a0_2260x722.jpeg)

Step 4: Apply number formatting
-------------------------------

Ctrl+shift+, to apply default number formatting

Ctrl+. to add decimals

![](https://www.financialmodellinghandbook.org/content/images/public/images/0aac0fb6-b4f2-4b98-8d93-7e911ab2a759_2230x726.jpeg)

Step 5: Add units
-----------------

Always, always, always add units straight away.

Lack of units or errors in units is a prime cause of error in modelling.

![](https://www.financialmodellinghandbook.org/content/images/public/images/cd6a8454-df87-419c-941b-92472592876e_2246x756.jpeg)

Remember - we’re not going to leave this here
---------------------------------------------

I find it helpful to quickly and easily add inputs where I am working, as I build up calculation blocks. It makes it easier and quicker to test calculations with different values. Don’t worry - we are not going to leave our model like this. All of our inputs will be relocated to the input sheet when we complete a section of modelling.

See [Modelling skill 10 - "How to Relocate Inputs."](https://www.financialmodellinghandbook.org/how-to-relocate-inputs/)
